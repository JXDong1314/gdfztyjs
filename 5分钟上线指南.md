## 📝 Step 1: 注册 GitHub（1分钟）

1. **访问**：https://github.com
2. **点击** `Sign up`
3. **填写**：
   - 邮箱
   - 密码
   - 用户名（这个很重要，会用在网址中）
4. **验证邮箱**

**✅ 完成！**

---

## 📦 Step 2: 创建仓库（1分钟）

1. **登录 GitHub** 后，点击右上角 **+** → **New repository**

2. **填写仓库信息**：
   ```
   Repository name: quiz-system
   （或者任何你喜欢的名字，建议用英文）
   
   Description: 通用技术刷题系统
   
   ⭕ Public（必须选公开）
   
   ✅ 勾选 "Add a README file"
   ```

3. **点击** `Create repository` 按钮

**✅ 仓库创建完成！**

记住你的仓库地址：
```
https://github.com/你的用户名/quiz-system
```

---

## 📤 Step 3: 上传文件（2分钟）

### 方式A：网页拖拽上传（最简单！）

1. **在仓库页面**，点击 **Add file** → **Upload files**

2. **打开你的项目文件夹**：
   ```
   C:\Users\Administrator\Desktop\通用技术刷题系统online
   ```

3. **全选所有文件**：
   - 按 `Ctrl + A`（全选）
   - **注意**：要选择文件夹里的内容，不是选择文件夹本身

4. **拖拽到浏览器的上传区**（显示"Drag files here"的地方）

5. **等待上传**（进度条显示）

6. **滚动到底部**，在 "Commit changes" 区域：
   ```
   输入: 初始上传
   ```

7. **点击** `Commit changes` 按钮

8. **等待完成**（可能需要2-5分钟，取决于网速）

**✅ 文件上传完成！**

### 方式B：分批上传（如果文件太多，一次上传失败）

1. **先上传主文件**：
   - 上传 `index.html`
   - 提交

2. **然后分别上传每个试卷文件夹**：
   - 上传 `2021年A卷` 文件夹
   - 提交
   - 上传 `2021年B卷` 文件夹
   - 提交
   - ... 依次上传所有文件夹

---

## 🚀 Step 4: 开启 GitHub Pages（30秒）

1. **在仓库页面**，点击顶部的 **⚙️ Settings**（设置）

2. **左侧菜单**，向下滚动找到 **Pages**

3. **在 "Build and deployment" 部分**：
   
   **Source（源）**: Deploy from a branch
   
   **Branch（分支）**:
   ```
   ▼ main  ▼ / (root)  [Save]
   ```
   - 第一个下拉选择：`main`
   - 第二个下拉选择：`/ (root)`
   - 点击 `Save` 按钮

4. **等待几秒**，页面会刷新，顶部会出现绿色提示框：
   ```
   ✅ Your site is live at https://你的用户名.github.io/quiz-system/
   ```

**✅ GitHub Pages 已开启！**

## 🔄 如何更新内容？

### 方法1：网页直接编辑

1. 在仓库中找到要修改的文件（如 `index.html`）
2. 点击文件，然后点击 ✏️（编辑按钮）
3. 修改内容
4. 点击 `Commit changes`
5. 等待 1-2 分钟，网站自动更新

### 方法2：上传新文件

1. 在仓库页面，进入对应文件夹
2. **Upload files**
3. 选择新文件（会自动覆盖同名文件）
4. 提交
5. 等待 1-2 分钟，网站自动更新

### 方法3：删除文件

1. 在仓库中找到要删除的文件
2. 点击文件，然后点击 🗑️（删除按钮）
3. 提交
4. 等待 1-2 分钟，网站自动更新

## 🐛 常见问题

### Q1: 显示 404 Not Found

**原因**：
- 网站还在构建中（首次需要5-10分钟）
- Pages 未正确开启

**解决**：
1. 等待 5-10 分钟
2. 刷新页面（按 `Ctrl + F5` 强制刷新）
3. 检查 Settings → Pages 是否已开启
4. 确认 `index.html` 已上传到根目录

### Q2: 图片显示不出来

**原因**：
- 图片文件夹未上传
- 路径错误

**解决**：
1. 检查是否所有试题文件夹都已上传
2. 在仓库页面查看文件结构，确认完整性
3. 浏览器按 `F12`，查看 Console 错误信息

### Q3: 样式丢失

**原因**：
- CSS/JS 文件未上传
- 缓存问题

**解决**：
1. 确认所有文件都已上传
2. 清除浏览器缓存
3. 使用无痕模式访问

### Q4: 更新后没变化

**原因**：
- GitHub Pages 缓存
- 浏览器缓存

**解决**：
1. 等待 2-3 分钟
2. 强制刷新：`Ctrl + F5`
3. 清除浏览器缓存

### Q5: 上传速度很慢

**原因**：
- GitHub 在国内访问较慢
- 文件太多

**解决**：
1. 分批上传（先上传主文件，再上传文件夹）
2. 压缩图片文件
3. 使用稳定的网络环境

## 🎯 重要提示

### ✅ 文件结构要求

**正确的结构**：
```
仓库根目录/
├── index.html          ← 必须在根目录！
├── 2021年A卷/
│   ├── 2021年A卷.md
│   └── *.png
├── 2021年B卷/
├── 2022年A卷/
└── ...
```

**❌ 错误的结构**：
```
仓库根目录/
└── 通用技术刷题系统online/  ← 不要把整个文件夹放进去！
    ├── index.html
    └── ...
```

### 📝 上传注意事项

1. **不要上传项目文件夹本身**，只上传文件夹**里面的内容**
2. `index.html` 必须在仓库根目录
3. 文件夹结构要保持原样
4. 确保所有图片都上传了

## 🌟 进阶功能

### 自定义域名（可选）

如果你有自己的域名（如 `quiz.com`）：

1. **在 GitHub**：
   - Settings → Pages
   - Custom domain 输入你的域名
   - Save

2. **在域名商**：
   - 添加 CNAME 记录
   - 指向：`你的用户名.github.io`

### 强制 HTTPS

1. Settings → Pages
2. ✅ 勾选 "Enforce HTTPS"
3. Save

### 查看访问统计

1. 仓库页面 → Insights
2. Traffic 查看访问量
