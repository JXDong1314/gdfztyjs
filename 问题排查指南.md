# 🔧 404 错误排查指南

## ❌ 错误信息
```
The requested URL '/' was not found on this server.
```

---

## 🔍 可能的原因

### 1️⃣ 默认域名已过期（最可能）

**症状**：
- 部署成功
- 但访问 `*.tcloudbaseapp.com` 显示域名过期错误
- 或者显示404

**解决方法**：
- ✅ **查看控制台中的实际访问域名**

---

## ✅ 立即解决方案

### 方案一：检查控制台中的访问域名

1. **访问控制台**
   ```
   https://console.cloud.tencent.com/tcb/env/index
   ```

2. **选择环境**: `gdfztyjs-2gpucv007e6d0c87`

3. **进入静态网站托管**
   - 点击左侧菜单 **静态网站托管**
   - 点击 **设置**

4. **查看域名信息**
   - 查找 **域名信息** 或 **访问域名**
   - 可能会显示：
     - 默认域名（可能已过期）
     - 测试域名
     - 或提示绑定自定义域名

5. **复制实际可用的访问域名**
   - 使用这个域名访问你的网站

---

### 方案二：检查静态网站配置

1. **在控制台检查**：
   - 静态网站托管 → 设置
   - 查看 **索引文档** 是否设置为 `index.html`
   - 查看 **错误文档** 设置（可选）

2. **如果未设置，请设置**：
   ```
   索引文档: index.html
   错误文档: index.html（可选）
   ```

---

### 方案三：重新部署确保文件正确

**双击运行**: `完整重新部署.bat`

这个脚本会：
1. 删除云端所有文件
2. 重新上传所有文件到根目录
3. 确保 index.html 在正确位置

---

### 方案四：手动检查和修复

#### Step 1: 检查云端文件

**双击运行**: `检查云端文件.bat`

查看是否有：
```
/index.html              ✅ 必须在根目录
/2021年A卷/              ✅ 文件夹
/2021年B卷/              ✅ 文件夹
...
```

**如果 index.html 不在根目录**，运行：
```
重新部署index.bat
```

---

### 方案五：使用备用访问方式

#### A. 通过文件直接访问

尝试访问：
```
https://gdfztyjs-2gpucv007e6d0c87.tcloudbaseapp.com/index.html
```

如果这样能访问，说明文件已上传，只是默认首页配置有问题。

#### B. 检查是否有其他域名

在控制台中，可能会提供：
- CDN加速域名
- 测试域名
- 临时访问域名

## 💡 最可能的情况

根据错误信息 `DEFAULT_DOMAIN_EXPIRED`，你遇到的是：

**默认域名已过期，但静态托管功能正常**

### 解决办法：

#### 选项A：绑定自定义域名（最佳）

如果你有域名（如 `yourdomain.com`）：

1. 控制台 → 静态网站托管 → 设置 → 域名管理
2. 添加自定义域名
3. 配置DNS解析
4. 等待生效（10分钟-24小时）

#### 选项B：使用腾讯云COS（推荐备选）

参考 `迁移到COS.md` 文档，COS提供：
- ✅ 永久稳定的访问域名
- ✅ 更大的免费额度
- ✅ 不会过期的问题
